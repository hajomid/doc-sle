A.3 December 2015 (Initial Release of SUSE Linux Enterprise Desktop 12 SP 1) #

General

  • Book “Subscription Management Tool for SLES 12 SP1” is now part of the documentation for SUSE Linux Enterprise Desktop.

  • Add-ons provided by SUSE have been renamed to modules and extensions. The manuals have been updated to reflect this change.

  • Numerous small fixes and additions to the documentation, based on technical feedback.

  • The registration service has been changed from Novell Customer Center to SUSE Customer Center.

  • In YaST, you will now reach Network Settings via the System group. Network Devices is gone (https://bugzilla.suse.com/show_bug.cgi?id=867809).

Chapter 3, System Recovery and Snapshot Management with Snapper

  • Added information about the new --sync switch for snapper delete to Section 3.5.4, “Deleting Snapshots” (Fate #317066).

  • Added Section 3.3.1, “Accessing and Identifying Snapshot Boot Entries” (Fate #317972 and Fate #318101).

  • Added a tip to Section 3.3, “System Rollback by Booting from Snapshots” on how to do a rollback to the initial installation state or the state before a system update (Fate #317973 and Fate #317900).

  • Added Section 3.1.3.3, “Creating and Mounting New Subvolumes” (Fate #318805, https://bugzilla.suse.com/show_bug.cgi?id=910602).

Chapter 5, Remote Access with VNC

  • Turned a note into a section, added information about VNC using secured protocol by default (Fate #318936), and removed tightvnc as it is fully replaced by tigervnc. All in Section 5.1.1, “Available Configurations”.

Chapter 4, Managing Software with Command Line Tools

  • Added Section 4.1.4, “Identifying Processes and Services Using Deleted Files” (Fate #318827).

  • Added more examples of zypper list-patches --cve in Section 4.1.3.1, “Installing Patches” (Fate #319053).

  • Added Section 4.1.2.2, “Installing Packages from Disabled Repositories” and a tip on removing all debuginfopackages in Section 4.1.2, “Installing and Removing Software with Zypper” (Fate #316287).

  • Added a sentence on announcing the need for the system reboot after specific patch is applied. (Fate #317872).

Chapter 11, journalctl: Query the systemd Journal

  • Added section Section 11.6, “Using YaST to Filter the systemd Journal” (Fate #318486).

Chapter 12, The Boot Loader GRUB 2

  • Updated/simplified the whole chapter to match the latest GRUB version, both command line and YaST version.

Chapter 13, UEFI (Unified Extensible Firmware Interface)

  • Added Section 13.1.4, “Using Non-Inbox Drivers” (Fate #317593).

Chapter 20, Basic Networking

  • Nanny is now on by default in Section 20.6.1.3, “Nanny” (Fate #318977).

Section 26.1, “Available Data Synchronization Software”

  • Mentioned cloud computing for file synchronization.

Chapter 31, Common Problems and Their Solutions

  • Improved GRUB 2 re-installation procedure in Section 31.6.2.4, “Modifying and Re-installing the Boot Loader”.

Part II, “System”

  • Added Chapter 17, Live Patching the Linux Kernel Using kGraft (Fate #313296 and Fate #313438).

Bugfixes

  • Removed obsolete acpid.service (https://bugzilla.suse.com/show_bug.cgi?id=918655).

  • Added a paragraph on secure boot enabled by default in Section 13.1.1, “Implementation on SUSE Linux Enterprise” (https://bugzilla.suse.com/show_bug.cgi?id=879486).

  • Removed documentation about VNC view-only passwords in Section 5.2, “Persistent VNC Sessions” because they are not available in SUSE Linux Enterprise Desktop (https://bugzilla.suse.com/show_bug.cgi?id=941307).

  • Fixed procedure to access the installed system in a rescue mode in Section 31.6.2.3, “Accessing the Installed System” (https://bugzilla.suse.com/show_bug.cgi?id=918217).

  • Added a new tip on updating the initramfs file after changing the default sysctl configuration in Section 9.2, “initramfs” (https://bugzilla.suse.com/show_bug.cgi?id=927506).

  • Added a tip on preventing wicked from deactivating the network device on NFS roots in Section 24.3.1, “Importing File Systems with YaST” and Section 20.4.1.2.5, “Activating the Network Device” (https://bugzilla.suse.com/show_bug.cgi?id=938152).

  • Fixed misleading statement about kernel-FLAVOR-extra in Section 30.5, “Support of Kernel Modules” (http://bugzilla.suse.com/show_bug.cgi?id=922976).

  • Btrfs/Snapper: Snapshots with new Subvolumes will not be Deleted ( https://bugzilla.suse.com/show_bug.cgi?id=910602).

  • Btrfs Documentation on Separate Subvolume on /var/lib and Supportability ( https://bugzilla.suse.com/show_bug.cgi?id=930424).

A.5 October 2014 (Initial Release of SUSE Linux Enterprise Desktop 12) #

General

  • Removed all KDE documentation and references because KDE is no longer shipped.

  • Removed all references to SuSEconfig, which is no longer supported (Fate #100011).

  • Move from System V init to systemd (Fate #310421). Updated affected parts of the documentation.

  • YaST Runlevel Editor has changed to Services Manager (Fate #312568). Updated affected parts of the documentation.

  • Removed all references to ISDN support, as ISDN support has been removed (Fate #314594).

  • Removed all references to the YaST DSL module as it is no longer shipped (Fate #316264).

  • Removed all references to the YaST Modem module as it is no longer shipped (Fate #316264).

  • Btrfs has become the default file system for the root partition (Fate #315901). Updated affected parts of the documentation.

  • The dmesg now provides human-readable time stamps in ctime()-like format (Fate #316056). Updated affected parts of the documentation.

  • syslog and syslog-ng have been replaced by rsyslog (Fate #316175). Updated affected parts of the documentation.

  • MariaDB is now shipped as the relational database instead of MySQL (Fate #313595). Updated affected parts of the documentation.

  • SUSE-related products are no longer available from http://download.novell.com but from http://download.suse.com. Adjusted links accordingly.

  • Novell Customer Center has been replaced with SUSE Customer Center. Updated affected parts of the documentation.

  • /var/run is mounted as tmpfs (Fate #303793). Updated affected parts of the documentation.

  • The following architectures are no longer supported: Itanium and x86. Updated affected parts of the documentation.

  • The traditional method for setting up the network with ifconfig has been replaced by wicked. Updated affected parts of the documentation.

  • A lot of networking commands are deprecated and have been replaced by newer commands (usually ip). Updated affected parts of the documentation.

    arp: ip neighbor
    ifconfig: ip addr, ip link
    iptunnel: ip tunnel
    iwconfig: iw
    nameif: ip link, ifrename
    netstat: ss, ip route, ip -s link, ip maddr
    route: ip route

  • Numerous small fixes and additions to the documentation, based on technical feedback.

Chapter 1, YaST Online Update

  • YaST provides an option to enable or disable the use of delta RPMs (Fate #314867).

  • Before installing patches that require a reboot, you are notified by YaST and can choose how to proceed.

Chapter 2, YaST in Text Mode

  • Added information on how to filter and select packages in the software installation module.

Chapter 3, System Recovery and Snapshot Management with Snapper

  • Updated the chapter and added new features (Fate #312751, Fate #316238, Fate #316233, Fate #316232, Fate #316222, Fate #316203, Fate #316222).

  • Added the section Section 3.3, “System Rollback by Booting from Snapshots” (Fate #316231, Fate #316221, Fate #316541, Fate #316522).

Chapter 5, Remote Access with VNC

  • The default VNC viewer is now tigervnc.

  • Added corrections on window manager start-up in persistent VNC sessions.

Chapter 4, Managing Software with Command Line Tools

  • Removed documentation about Zypper's rug compatibility mode (Fate #317708)

  • Rewrote Section 4.1.6, “Querying Repositories and Packages with Zypper”.

Chapter 9, Booting a Linux System

  • Significantly shortened the chapter, because System V init has been replaced by systemd. systemd is now described in a separate chapter: Chapter 10, The systemd Daemon.

Chapter 10, The systemd Daemon

  • Added a new chapter on systemd and the YaST Services Manager (Fate #316631, Fate #312568).

  • New section on loading kernel modules (http://bugzilla.suse.com/show_bug.cgi?id=892349).

Chapter 11, journalctl: Query the systemd Journal

Chapter 12, The Boot Loader GRUB 2

  • Added new section Section 12.4, “Differences in Terminal Usage on z Systems”.

Chapter 13, UEFI (Unified Extensible Firmware Interface)
